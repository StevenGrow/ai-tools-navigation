# 响应式布局测试指南

本指南详细说明如何使用响应式布局测试工具来验证 AI 工具导航网站在不同设备和屏幕尺寸下的显示效果。

## 📋 测试概述

响应式布局测试包括以下几个方面：

### 🖥️ 桌面端测试 (>1024px)
- 1920×1080 (1080p 显示器)
- 1366×768 (笔记本常见分辨率)
- 1200×800 (最小桌面端)

### 📱 平板端测试 (768px-1024px)
- 1024×768 (iPad 横屏)
- 768×1024 (iPad 竖屏)
- 800×1280 (Android 平板)

### 📱 手机端测试 (≤768px)
- 375×667 (iPhone SE)
- 414×896 (iPhone 11)
- 360×640 (Android 常见)
- 320×568 (最小手机屏幕)

## 🚀 快速开始

### 方法一：使用测试页面

1. 打开 `test-responsive.html` 文件

3. 点击相应按钮测试不同设备类型

## 🎯 测试标准

### 移动端触摸友好性标准
- **最小触摸尺寸**: 44×44px (Apple HIG 推荐)
- **输入框最小高度**: 44px
- **按钮最小高度**: 44px
- **复选框最小尺寸**: 18×18px

### 响应式断点
```css
/* 桌面端 */
@media (min-width: 1025px) { /* 并排布局 */ }

/* 平板端 */
@media (max-width: 1024px) { /* 单列布局 */ }

/* 手机端 */
@media (max-width: 768px) { /* 移动端优化 */ }

/* 小屏手机 */
@media (max-width: 480px) { /* 单列工具卡片 */ }
```

### 性能标准
- **布局稳定性**: 无意外的布局跳动
- **触摸响应**: 操作反馈及时
- **视觉层次**: 重要元素突出显示

## 🐛 常见问题及解决方案

### 1. 工具卡片在移动端显示异常
**问题**: 卡片过小或布局错乱
**解决**: 检查 CSS Grid 媒体查询设置

```css
@media (max-width: 480px) {
    .tools-grid {
        grid-template-columns: 1fr; /* 单列显示 */
    }
}
```

### 2. 按钮在移动端难以点击
**问题**: 按钮尺寸小于 44px
**解决**: 增加按钮最小尺寸

```css
@media (max-width: 768px) {
    .tool-action-btn {
        min-width: 44px;
        min-height: 44px;
        padding: 0.6rem 0.8rem;
    }
}
```

### 3. 模态框在小屏幕上显示不全
**问题**: 模态框超出视口
**解决**: 设置响应式尺寸

```css
@media (max-width: 768px) {
    .modal-content {
        width: 95%;
        max-height: 90vh;
        overflow-y: auto;
    }
}
```

### 4. 导航项在移动端过于拥挤
**问题**: 导航项间距不足
**解决**: 调整间距和换行

```css
@media (max-width: 768px) {
    .category-nav {
        gap: 0.8rem;
        padding: 0.8rem 1rem;
    }
    
    .nav-item {
        padding: 0.4rem 0.8rem;
        min-height: 44px;
    }
}
```
